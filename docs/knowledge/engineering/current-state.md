# Notes

- `cargo nextest run -p jellyflow-runtime -p jellyflow-egui -p jellyflow-proof --lib --no-fail-fast` now runs 529 tests across runtime, egui, and proof.
- Low-zoom visual confidence was checked through gallery snapshots; the automation builder now degrades node internals instead of overlapping Dify-style controls, while shader and ERD examples keep readable anchors and wires.
- Subagent research on xyflow, egui-snarl, Rete, Unreal Blueprint, Unity Shader Graph, Dify, and mind-map products reinforced the current boundary: headless owns semantic node kits, typed ports, slots, anchors, chrome, measurement, validation, and action/menu descriptors; adapters own widgets, focus, menus, layout measurement, and local component libraries. The Node UI Kit Component Contract is now captured as the next design layer; the remaining design gap is a formal adapter capability matrix, not a shared widget crate.
- The Node UI Authoring Contracts plan is implemented as semantic contracts, not a shared widget crate. GPUI now has a first-class adapter crate and a live `measured_element` consumer; capability reporting must remain coverage-gated so projection/partial reports do not imply full layout-pass measurement.
- The current plan intentionally narrows maturity work to open-gpui only. egui and Dioxus should not be expanded in this stage unless shared runtime compatibility requires it.
- Review gate outcome: local code review found and fixed one real issue in the GPUI example: repeated identical layout-pass facts originally incremented `measurement_revision`, which would make `NodeGraphStore::report_node_measurement` report changed every frame. The fix reuses the existing revision when geometry is unchanged and fresh, while dirty/missing or changed geometry still bumps revision. A regression test covers the no-refresh path.
- Open GPUI authoring facade cleanup refined ownership: `jellyflow-open-gpui` owns binding, live-store authoring planning, semantic repeatable action mapping, scoped element ids, renderer resolution/fallback, and generic host-context services; `canvas-jellyflow` owns concrete Open GPUI widgets, weak-entity dispatch, focus/popup behavior, demo defaults, visual layout, and refresh notifications.
- Final review found and fixed one adapter-id issue: dynamic element-id segments are now escaped so semantic keys containing `:` cannot collide, and the custom renderer badge id is node scoped so multiple same-renderer nodes do not reuse the same local id.
- Open GPUI node component kit implementation is complete through committed slices: `jellyflow-open-gpui` has product fixture catalogs, renderer preset/report contracts, host visual interaction gates, and dynamic repeatable lifecycle gates; `canvas-jellyflow` has a host-local `node_component_kit`, Dify/shader/ERD/mind-map product renderers, a switchable product gallery, repeatable port diagnostics, and optional screenshot smoke export. Runtime/core/layout remain free of GPUI widgets.
- Open GPUI product gallery UX stabilization keeps the same boundary: runtime owns semantic fixture facts, `jellyflow-open-gpui` owns widget-free report contracts, and `canvas-jellyflow` owns concrete GPUI component layout and event policy. The latest strict gate treats unreadable clipped content and fixture node overlap as failures.
- The Product Interactions plan keeps the same architecture but treats shader drag, clickable ports/edges, Connect/reconnect tool UX, dropped-wire insertion, and component readability as product gates. Subagent review found this is mostly an Open GPUI adapter/canvas integration maturity gap, not a reason to move widget state into runtime.
- Product Interactions U8 promotes the concrete Open GPUI interaction baseline into `jellyflow-open-gpui::testing::assert_product_interaction_report_gates`. The hard structured baseline now requires full drag pointer sequences, control shielding, port hotspot reachability, visible tool switching, connect/reconnect/dropped-wire store sync, and repeatable overflow indicators. Screenshot export remains a smoke/review aid only.
- Native review found product nodes could move only a little and then stop. Root cause: the Open GPUI example synced canvas transforms into `NodeGraphStore` on pointer move and immediately rebuilt `CanvasEditor`, which dropped `ToolState::Translating`. The example now defers editor refresh while pointer interactions are active and refreshes after idle; `canvas_view_keeps_drag_state_while_syncing_product_surface_moves` covers the real `JellyflowCanvasView` path.
- The next plan is intentionally Open GPUI-first, not a cross-framework widget library. It compares egui-snarl and React Flow-style affordances against the current Open GPUI substrate, then turns the real gaps into generic canvas interaction work, widget-free adapter evidence, and host-local product component layout.
- Canvas Node UI Foundations U4 moved connection lifecycle events into generic Open GPUI canvas: `CanvasEditor::take_connection_release()` now reports connected, reconnected, dropped, and rejected releases; connection previews carry free/valid/invalid target feedback; port hit budgets are larger; selected reconnect handles share the same 20px target size; `canvas-jellyflow` consumes dropped releases instead of inferring dropped-wire intent from raw pointer-up events.
- Canvas Node UI Foundations U5 moved graph affordance proof into `jellyflow-open-gpui`: `OpenGpuiGraphAffordanceEvidence` records committed route family, preview policy, port placement budget, endpoint/reconnect hit budgets, drag regions, and readable layout regions. `assert_product_interaction_report_gates` now fails when previews fall back to direct lines or when graph affordance evidence is missing.
- Canvas Node UI Foundations U6/U7 moved the Open GPUI product renderers from fixed absolute rows toward host-local adaptive layout primitives. `node_component_kit` owns `AdaptiveNodeLayoutStack` and repeatable-list planning; Dify/shader/ERD/mind-map renderers derive their product regions from those plans and keep reduced-size cards inside node bounds with explicit overflow indicators.
- Current capability story: Open GPUI is the only mature adapter target in this stage. egui and proof still validate shared headless contracts, but docs should not claim mature egui/Dioxus parity for the new Open GPUI canvas interaction foundation.
- U8/U9 closeout updated README, regression docs, decisions, log, and current-state memory with the final ownership boundary: generic canvas graph UX in Open GPUI canvas, widget-free adapter presets/reports in `jellyflow-open-gpui`, and host-local component/layout primitives in `canvas-jellyflow`.
- Native UX Polish U1-U6 has landed in commits `a2030fc`, `d21726a`, `2abc24a`, `0346506`, and `e3806a1` in the Jellyflow repo, with matching local `repo-ref/open-gpui` commits `a711328`, `7085d86`, `2261be8`, `c6a8bd1`, `2773dcd`, `1f97a81`, and `aa274d8`. The current gate story is structured evidence first: native lifecycle evidence, screenshot ROI evidence, graph affordance evidence, full reconnect sequence evidence, and measured internals evidence. Screenshot PNGs and native launch review remain aids, not correctness oracles.
- Native UX Polish U7 closed the documentation and memory loop, and final verification repeated the broad gates after the last assertion alignment fix. The plan is now complete from the repository's perspective; `repo-ref/open-gpui` remains a local main fork ahead of origin and was not pushed.
- The 005 XYFlow-style node component refactor corrects the previous fit-estimation evidence direction. Active contracts now use `OpenGpuiMeasuredInternalsEvidence`, graph-affordance-only `OpenGpuiSurfaceStyleBudget`/`OpenGpuiStyleBudgetEvidence`, example-local `OpenGpuiNodeComponentRegistry`/context mapping, component-owned measured handles/regions, and graph-owned connection/reconnect behavior. Adapter-owned arbitrary text/control fit heuristics must not return as public API.
- Component Host Seam U1-U3 replaces the temporary example-local component registry with the existing widget-free renderer registry plus one concrete host render table. `canvas-jellyflow` now has an explicit `render_open_gpui_node_wrapper` path for custom and fallback renderers, product components receive `OpenGpuiNodeComponentContext`, and repeatable/static anchors are declared through measured handle/anchor helpers instead of hidden 1x1 or offscreen layout hacks.
- Component Host Seam U4/U5 keeps size and interaction semantics explicit. The host-local `OpenGpuiNodeSizePolicy` derives fixed/intrinsic/resizable projection from runtime surface budgets, preserves user-resized dimensions under `Resizable`, and clamps below min readable budgets. Control rows/chips now emit both measured control and drag-exclusion regions; overflow indicators are component-declared measured regions. The removed `repeatable_label_needs_overflow_badge` path should not return because arbitrary text length is not a library contract.
- Component Host Seam U6-U8 boundary decision: do not force ordinary port connect/reconnect through menu/action surfaces. The mature split is XYFlow-like: direct graph gestures own continuous pointer lifecycle and connection/reconnect planning; semantic action/menu plans own discrete product choices such as dropped-wire insert, node menus, inspector actions, toolbar actions, and blackboard mutations. Current builtin fixtures have no useful Graph-target action, so adding an empty graph menu would be worse than documenting the boundary.
- Open GPUI example-boundary migration moves the concrete product gallery from the former Open GPUI workspace example path to `examples/canvas-jellyflow`. The Open GPUI release workspace should retain generic canvas/framework primitives only; Jellyflow owns the product gallery, while `jellyflow-open-gpui` remains widget-free. Until `open-gpui-*` 0.2 crates are published, the example uses a pinned Open GPUI git revision instead of local `repo-ref` path dependencies.
- Open GPUI adapter productization is committed in root through `b792c3a`, `8cba813`, and `45460fe`, with local `repo-ref/open-gpui` main commits through `5b890c4`. The key U4 correction is that shader product route evidence now reports orthogonal committed routes, and `canvas-jellyflow` proves actual connect-preview route parity instead of trusting renderer presets.
- Open GPUI interaction determinism first slice follows the XYFlow/egui-snarl boundary: runtime stays headless, generic Select/Connect pointer priority and endpoint policy live in `repo-ref/open-gpui/crates/canvas`, widget-free proof remains in `jellyflow-open-gpui`, and concrete GPUI product roles/measurement scheduling live in `canvas-jellyflow`. See the 2026-07-05 progress, verification, and subagent memory concepts for details.
- Open GPUI interaction determinism U5-U7 closes the remaining structured product gates. `jellyflow-open-gpui` now requires visible port markers, measured anchors, canvas handles, hit-test endpoints, edge endpoints, first-pointer ownership, control shielding, no-pointer readiness, and connection release/drop evidence. `canvas-jellyflow` proves those facts from the real product projection path; disabled-port evidence uses an unconnected disabled product port so the probe checks non-connectability without corrupting an existing edge.
