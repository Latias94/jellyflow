# JNR-030 - Direction And Origin Resize Planning

Date: 2026-06-02

Extended `runtime::resize` beyond size-only bottom/right planning.

Result:

- Added `NodeResizeDirection` for XyFlow-style control directions.
- Added `NodeResizeContext` and `plan_node_resize_with_context`.
- Updated store planning to use resolved global `node_origin`.
- Added `from_pos` and `to_pos` to resize plans.
- Emitted `SetNodePos` before `SetNodeSize` for left/top controls.
- Covered left-edge resize, global origin fallback, and per-node origin override behavior.

Split decision:

- Parent/child extent and keep-aspect-ratio parity are deferred because XyFlow derives those clamps
  from pointer start values and distance clamps. The current Jellyflow request is target-size based,
  so exact parity should wait for a pointer-resize session request or adapter evidence.

Validation:

- `cargo fmt --check`
- `cargo nextest run -p jellyflow-runtime resize`
- `cargo nextest run -p jellyflow-runtime drag_parent_expansion`
- `cargo nextest run -p jellyflow-runtime --test public_surface`
- `cargo clippy -p jellyflow-runtime --all-targets -- -D warnings`
