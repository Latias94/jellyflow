# JPB-040 - Policy-Backed Connect And Reconnect Planning

Date: 2026-05-30

## Changes

- Added policy-aware connect planning variants and made default connect planners resolve persisted
  node/port connectability through `runtime::policy`.
- Added policy-aware reconnect planning variants and made default reconnect planners resolve edge
  reconnectability plus final endpoint connectability through `runtime::policy`.
- Added tests for disabled node/port connectability and disabled/endpoint-specific reconnectability.

## Concern

Delete enforcement was not changed because there is no runtime delete planner path to route through
the facade. `runtime::policy` exposes node/edge `deletable` state for adapters; a delete planner
should be split into a follow-on if the runtime should own that behavior.
